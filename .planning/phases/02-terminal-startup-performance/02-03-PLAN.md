<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IPC → lazy getters → manager instances | IPC handlers now go through getter functions; managers are not pre-allocated |
| Packaged app → DevTools gate | app.isPackaged prevents renderer internals from being accessible in production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Information Disclosure | DevTools in production | mitigate | app.isPackaged guard prevents openDevTools() in packaged builds. Eliminates renderer inspection surface for end users in distributed builds. This is the mitigation itself. |
| T-02-09 | Elevation of Privilege | lazy getter with null mainWindow | mitigate | All getters use `mainWindow!` non-null assertion. mainWindow is set before registerIpcHandlers() is called, so it is always non-null when any IPC handler fires. If window is closed and a stale IPC arrives, the manager is already disposed and the getter would create a new manager with a destroyed window — mitigated by the existing `isDestroyed()` checks inside each manager. |
| T-02-10 | Denial of Service | missed dispose on lazy manager | accept | If a manager is never initialized (user never opens a terminal), its dispose call in the closed handler is `null?.disposeAll()` via optional chaining — no-op. Accepted: no resource leak because the manager was never allocated. |
| T-02-11 | Tampering | flushSnapshot ordering | mitigate | flushSnapshot() is called FIRST in the closed handler before stateStore = null. Order is enforced in the implementation. If stateStore is somehow null (defensive), optional chaining prevents a crash. |
</threat_model>
