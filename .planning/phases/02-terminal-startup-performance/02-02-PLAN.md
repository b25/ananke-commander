<objective>
Debounce electron-store writes in StateStore by accumulating setSnapshot patches in memory and flushing after 300ms of inactivity.

Purpose: Rapid pane interactions (switching, resizing, typing) fire state:set IPC rapidly. Without debounce, each call triggers synchronous JSON serialization and disk write in electron-store, causing write storms. A single 300ms debounce collapses burst writes into one (PERF-06, D-04).

Output:
- stateStore.ts — pendingPatch accumulator, debounced setSnapshot, merged getSnapshot, public flushSnapshot() method

**Step 1 — Add two private fields to the StateStore class** (add immediately after the existing `private store: Store<StoreSchema>` field declaration):
```typescript
private pendingPatch: Partial<AppStateSnapshot> = {}
private debounceTimer: ReturnType<typeof setTimeout> | null = null
```

**Step 2 — Replace setSnapshot** with a debounced implementation:
```typescript
setSnapshot(patch: Partial<AppStateSnapshot>): void {
  Object.assign(this.pendingPatch, patch)
  if (this.debounceTimer) clearTimeout(this.debounceTimer)
  this.debounceTimer = setTimeout(() => {
    this.flushSnapshot()
  }, 300)
}
```

**Step 3 — Add a new public flushSnapshot() method** (add immediately after setSnapshot):
```typescript
flushSnapshot(): void {
  if (this.debounceTimer) {
    clearTimeout(this.debounceTimer)
    this.debounceTimer = null
  }
  const patch = this.pendingPatch
  this.pendingPatch = {}
  if (patch.workspaces !== undefined) this.store.set('workspaces', patch.workspaces)
  if (patch.activeWorkspaceId !== undefined) this.store.set('activeWorkspaceId', patch.activeWorkspaceId)
  if (patch.settings !== undefined) this.store.set('settings', patch.settings)
  if (patch.recentlyClosed !== undefined) this.store.set('recentlyClosed', patch.recentlyClosed)
}
```

**Step 4 — Update getSnapshot** to merge pending patch over disk values (prevents stale reads during debounce window):
```typescript
getSnapshot(): AppStateSnapshot {
  const disk: AppStateSnapshot = {
    workspaces: this.store.get('workspaces'),
    activeWorkspaceId: this.store.get('activeWorkspaceId'),
    settings: this.store.get('settings'),
    recentlyClosed: this.store.get('recentlyClosed')
  }
  return { ...disk, ...this.pendingPatch }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IPC → stateStore.setSnapshot | Renderer sends patch objects via state:set IPC; main process writes to disk |
| stateStore.getSnapshot → IPC response | Main process reads state and returns to renderer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information Disclosure | stateStore.ts pendingPatch | accept | pendingPatch holds in-memory app state (workspace layout, settings). This is the same data already in electron-store. No new exposure — same process memory. Low risk. |
| T-02-05 | Denial of Service | debounce timer leak | mitigate | flushSnapshot() clears debounceTimer before flushing. If called repeatedly without setSnapshot calls, it is a no-op (pendingPatch is empty, no disk write). Timer cleared on flush, preventing accumulation. |
| T-02-06 | Tampering | stale state read | mitigate | getSnapshot merges pendingPatch over disk values. Renderer always receives the latest in-memory state including unflushed patches. Prevents UI "jumping" from stale reads. |
| T-02-07 | Denial of Service | state loss on quit | mitigate | Plan 03 wires flushSnapshot() into the window 'closed' handler. If Plan 03 is not yet executed, there is a 300ms window on quit where state could be lost — this is a known gap until Plan 03 closes it. |
</threat_model>
