Rewrite `src/main/pty/terminalManager.ts` to add per-session output buffering. Each spawned PTY pushes data into a `string[]` buffer keyed by paneId. A single `setInterval` at 8ms flushes all non-empty buffers in one pass. The interval starts on first spawn and stops when no sessions remain.

Add these private fields to the class (after `private procs`):
```typescript
private buffers = new Map<string, string[]>()
private flushTimer: ReturnType<typeof setInterval> | null = null
```

Add these two private methods (after the constructor):
```typescript
private startFlushing(): void {
  if (this.flushTimer) return
  this.flushTimer = setInterval(() => this.flushAll(), 8)
}

private stopFlushing(): void {
  if (this.flushTimer) {
    clearInterval(this.flushTimer)
    this.flushTimer = null
  }
}

private flushAll(): void {
  for (const [paneId, chunks] of this.buffers) {
    if (chunks.length === 0) continue
    const data = chunks.join('')
    this.buffers.set(paneId, [])
    if (!this.win.isDestroyed() && !this.win.webContents.isDestroyed()) {
      this.win.webContents.send('pty:data', { paneId, data })
    }
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PTY process → IPC → renderer | PTY stdout/stderr is opaque byte stream from the shell process; sent to renderer via Electron IPC |
| Renderer → contextBridge → IPC | Renderer calls pty:write which routes keystrokes to PTY stdin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | terminalManager.ts buffers | mitigate | Buffer contains raw PTY output strings. A runaway process producing infinite output could grow the buffer between 8ms flushes. Mitigation: flushAll() is called every 8ms so maximum buffer growth per session is bounded by PTY throughput over 8ms — acceptable. No per-session buffer size cap needed at this throughput (typical PTY max is ~1MB/s, 8ms = ~8KB max). |
| T-02-02 | Information Disclosure | useXterm.ts WebGL context | accept | WebGL context loss handling disposes the addon silently. No sensitive data is exposed via the context loss event. Low-value target, no PII in terminal rendering state. |
| T-02-03 | Denial of Service | setInterval leak | mitigate | stopFlushing() clears the interval when procs.size === 0. disposeAll() calls dispose() for each session which decrements procs size. Interval will stop when all sessions are closed. |
</threat_model>
